# 背包系统预制体配置完整指南

## 快速开始（5步完成）

### 第1步：创建文件夹结构

在Unity项目中创建以下文件夹：
```
Assets/
├── Prefabs/
│   └── UI/
│       ├── BackpackButton.prefab
│       ├── BackpackPanel.prefab
│       ├── FurnitureItem.prefab
│       └── FurnitureEditController.prefab
├── Resources/
│   └── Furniture/
│       ├── chair_001.asset
│       ├── table_001.asset
│       └── ...
└── Scripts/
    └── Editor/
        ├── BackpackButtonAutoSetup.cs
        ├── BackpackPanelAutoSetup.cs
        ├── FurnitureItemAutoSetup.cs
        └── FurnitureEditControllerAutoSetup.cs
```

### 第2步：使用编辑器工具创建预制体

#### 2.1 创建背包按钮
1. 在场景中创建一个空物体，命名为 `BackpackButtonSetup`
2. 挂载脚本 `BackpackButtonAutoSetup.cs`
3. （可选）在Inspector中指定一个背包图标Sprite
4. 勾选 `Auto Setup`
5. 将创建的 `BackpackButton` 拖到 `Assets/Prefabs/UI/` 保存为预制体
6. 删除场景中的临时物体

#### 2.2 创建背包面板
1. 在场景中创建一个空物体，命名为 `BackpackPanelSetup`
2. 挂载脚本 `BackpackPanelAutoSetup.cs`
3. 勾选 `Auto Setup`
4. 将创建的 `BackpackPanel` 拖到 `Assets/Prefabs/UI/` 保存为预制体
5. **重要**：手动配置以下引用：
   - `Outfit Item Prefab`：稍后创建
   - `Furniture Item Prefab`：稍后创建
6. 删除场景中的临时物体

#### 2.3 创建家具列表项
1. 在场景中创建一个空物体，命名为 `FurnitureItemSetup`
2. 挂载脚本 `FurnitureItemAutoSetup.cs`
3. （可选）指定默认图标Sprite
4. 勾选 `Auto Setup`
5. 将创建的 `FurnitureItem` 拖到 `Assets/Prefabs/UI/` 保存为预制体
6. 删除场景中的临时物体

#### 2.4 创建家具编辑控制器
1. 在场景中创建一个空物体，命名为 `FurnitureEditControllerSetup`
2. 挂载脚本 `FurnitureEditControllerAutoSetup.cs`
3. 勾选 `Auto Setup`
4. 将创建的 `FurnitureEditController` 拖到 `Assets/Prefabs/UI/` 保存为预制体
5. 删除场景中的临时物体

#### 2.5 更新背包面板引用
1. 打开 `BackpackPanel` 预制体
2. 在 `BackpackPanel` 脚本上配置：
   - `Outfit Item Prefab` → 拖入 `OutfitItem.prefab`（如果有的话，或使用现有装扮项预制体）
   - `Furniture Item Prefab` → 拖入 `FurnitureItem.prefab`
3. 保存预制体

### 第3步：创建示例家具数据

1. 在 `Assets/Resources/Furniture/` 文件夹中：
2. 右键 → `绒毛几何物语` → `家具数据`
3. 创建3-5个示例家具：

**示例1 - 云朵沙发**
- 家具ID: `sofa_cloud_001`
- 家具名称: 云朵沙发
- 描述: 软绵绵的云朵沙发，坐上去就像飘在天上
- 分类: Seat
- 默认缩放: 1.0
- 缩放范围: 0.7 - 1.5
- 是否可翻转: 是

**示例2 - 几何茶几**
- 家具ID: `table_geo_001`
- 家具名称: 几何茶几
- 描述: 简洁的几何造型茶几
- 分类: Table
- 默认缩放: 1.0
- 缩放范围: 0.8 - 1.3
- 是否可翻转: 否

**示例3 - 毛线球吊灯**
- 家具ID: `light_yarn_001`
- 家具名称: 毛线球吊灯
- 描述: 温暖的毛线球造型吊灯
- 分类: Light
- 默认缩放: 1.0
- 缩放范围: 0.5 - 1.2
- 是否可翻转: 否

### 第4步：配置场景中的HomeManager

1. 打开 `HomeScene` 场景
2. 找到 `HomeManager` 物体（或创建一个并挂载脚本）
3. 在Inspector中配置：

| 字段 | 值 |
|------|-----|
| 主菜单场景名称 | MainScene |
| 玩家角色预制体 | PlayerPrefab |
| 玩家出生位置 | 指定Transform |
| 虚拟摇杆 | 场景中的Joystick |
| **背包按钮预制体** | `BackpackButton.prefab` |
| **背包按钮父节点** | 留空（自动使用Canvas） |
| **家具编辑控制器预制体** | `FurnitureEditController.prefab` |

### 第5步：测试运行

1. 点击Play
2. 进入家园场景
3. 检查左上角是否有背包按钮
4. 点击背包按钮，检查：
   - 是否弹出背包面板
   - Tab切换是否正常
   - 家园装扮Tab是否显示家具列表
   - 点击家具项是否显示"装饰"按钮
   - 点击"装饰"是否进入编辑模式
   - 编辑工具栏是否显示（翻转、缩放、确认）
   - 拖拽、翻转、缩放是否正常
   - 确认后是否保存家具位置
   - 背包是否自动重新弹出

## 详细配置说明

### 背包按钮预制体结构

```
BackpackButton (GameObject)
├── RectTransform: 锚点(0,1)，尺寸(80,80)
├── Image: 背包图标或背景色
├── Button: 点击事件
├── BackpackButton.cs: 脚本
└── NewItemBadge (子物体)
    ├── RectTransform: 右上角
    └── Image: 红色圆点
```

### 背包面板预制体结构

```
BackpackPanel (GameObject)
├── RectTransform: 全屏
├── Image: 白色半透明背景
├── BackpackPanel.cs: 脚本
├── Title (Text): "背包"
├── CloseBtn (Button): 关闭按钮
├── TabContainer (Horizontal Layout)
│   ├── CharacterTab (Button): 主角装扮
│   └── FurnitureTab (Button): 家园装扮
├── CharacterContent (ScrollView): 主角装扮列表
│   └── Viewport/Content (Vertical Layout)
└── FurnitureContent (ScrollView): 家具列表
    └── Viewport/Content (Vertical Layout)
```

### 家具列表项预制体结构

```
FurnitureItem (GameObject)
├── RectTransform: 宽度500，高度100
├── Image: 白色背景
├── FurnitureItemUI.cs: 脚本
├── SelectedBorder (Image): 选中态边框（默认隐藏）
├── Icon (Image): 家具图标
├── Name (Text): 家具名称
└── DecorateBtn (Button): 装饰按钮（默认隐藏）
    └── Text: "装饰"
```

### 家具编辑控制器预制体结构

```
FurnitureEditController (GameObject)
├── FurnitureEditController.cs: 脚本
└── ToolbarPanel (Panel): 工具栏
    ├── RectTransform: 居中，尺寸(400,80)
    ├── Image: 深色半透明背景
    ├── HorizontalLayoutGroup: 水平布局
    ├── FlipBtn (Button): 翻转
    ├── ScaleSliderContainer
    │   ├── ScaleSlider (Slider): 缩放滑条
    │   └── ScaleValue (Text): 缩放值显示
    └── ConfirmBtn (Button): 确认
```

## 常见问题

### Q: 背包按钮没有显示？
A: 检查：
1. HomeManager中是否正确配置了背包按钮预制体
2. 场景中是否有Canvas
3. 按钮是否在Canvas的渲染范围内

### Q: 家具列表为空？
A: 检查：
1. Resources/Furniture/ 目录下是否有家具数据文件
2. 家具数据的 `isUnlocked` 是否为 true
3. BackpackPanel脚本是否正确引用了FurnitureItem预制体

### Q: 编辑家具时无法拖拽？
A: 检查：
1. 家具实例是否有Collider2D
2. 相机是否正确设置为MainCamera
3. 是否有其他UI遮挡

### Q: 确认后家具没有保存？
A: 检查：
1. HomeManager的 `PlaceFurniture` 方法是否被调用
2. 家具是否正确添加到decorations列表
3. PlayerPrefs是否正确保存

### Q: 背包重新打开时滚动位置没有恢复？
A: 检查：
1. BackpackPanel是否正确记录了 `furnitureScrollPos`
2. Reopen方法是否被正确调用
3. ScrollRect的normalizedPosition是否正确设置

创建 `CraftManager.cs`：
